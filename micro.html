Microservices Overview

What are Microservices?

Microservices architecture is an approach to developing a single application as a suite of small services, each running in its own process and communicating with lightweight mechanisms.

  • Each service implements a specific business capability
  • Services can be developed, deployed, and scaled independently
  • Improved fault isolation and resilience
  • Technology diversity - each service can use different technologies

Benefits in this Project

The Inception project implements microservices using Docker containers to demonstrate modern application architecture principles.

  • Isolation of concerns (database, application, web server)
  • Independent scaling of services based on demand
  • Faster development cycles for individual components
  • Easier maintenance and updates

Communication Flow

  1. Client requests reach NGINX on port 443 (HTTPS)
  2. NGINX routes requests based on URL patterns
  3. WordPress handles dynamic content requests
  4. Static portfolio serves static website content
  5. MariaDB provides data persistence for WordPress
  6. Redis caches frequently accessed data
  7. Adminer provides database management interface
  8. FTP server allows file management
  9. Portainer provides container management UI

Network Architecture

  • All services connected via custom inception network
  • Bridge network driver provides internal DNS resolution
  • External ports exposed only for necessary services
  • Internal communication uses service names as hostnames
  • Secure communication with TLS encryption

Data Flow

  • Persistent volumes for database, WordPress, and Redis data
  • Secrets management for sensitive configuration
  • Environment variables for service configuration
  • Dependency management with depends_on

Networking & Communication

Docker Network Configuration

Custom Bridge Network

The project uses a custom bridge network named inception to facilitate communication between containers.

Network Configuration:
networks:
  inception:
    driver: bridge

Service Discovery

Docker provides automatic DNS resolution for containers on the same network using service names as hostnames.

Internal Communication:
  • WordPress can connect to MariaDB using hostname mariadb
  • NGINX can proxy requests to WordPress using wordpress
  • Adminer can manage MariaDB using mariadb

Deployment & Management

Docker Compose Configuration

The entire microservices architecture is defined and managed using Docker Compose, which simplifies deployment and orchestration.

Key Features

  • Service Dependencies: depends_on ensures correct startup order
  • Volume Management: Persistent data storage for databases and files
  • Secrets Management: Secure handling of passwords and sensitive data
  • Environment Variables: Configuration externalization
  • Restart Policies: on-failure for resilience

Deployment Commands

# Build and start all services
$ docker-compose up -d

# View running services
$ docker-compose ps

# View logs
$ docker-compose logs

# Stop services
$ docker-compose down

# Rebuild specific service
$ docker-compose up -d --build nginx

Important Notes

  • Ensure Docker and Docker Compose are installed on the host system
  • Create necessary directories for volume mounting (/home/ien-niou/data/)
  • Set up proper secrets files with secure passwords
  • Configure environment variables in .env file
  • Obtain SSL certificates for HTTPS configuration
